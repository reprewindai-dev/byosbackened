API INTEGRATION (REQUIRED BACKEND ENDPOINTS)
================================================================================

The component calls 6 REST endpoints (all auto-include Bearer token auth):

1. GET /api/v1/stripe/subscription
   Returns: Current plan ID, status, next billing date

2. GET /api/v1/stripe/plans
   Returns: Available plans (optional - component has fallback)

3. POST /api/v1/stripe/checkout
   Input: { plan_id: string, billing_cycle: "monthly" }
   Returns: { checkout_url: string }

4. GET /api/v1/stripe/session/{session_id}
   Returns: { status: "complete|pending", ... }
   Used for: Payment verification after Stripe redirect

5. POST /api/v1/stripe/portal
   Returns: { portal_url: string }
   Used for: Opening Stripe billing portal in new tab

6. GET /api/v1/stripe/invoices
   Returns: { invoices: [...] }
   Each invoice needs: id, number, date, amount, status, pdf_url

Full API specification available in: /apps/dashboard/BILLING_IMPLEMENTATION.md

TECHNICAL SPECIFICATIONS
================================================================================

ARCHITECTURE
- React functional component with hooks
- useEffect for data loading and polling
- useCallback for memoized callbacks
- useRef for polling state and timers
- axios via /utils/api.js for HTTP

STATE MANAGEMENT
- subscription: Current plan and status
- plans: Available plans (fallback to component defaults)
- invoices: Invoice history
- loading: Initial page load
- error: Error message
- upgrading: Current upgrade in progress (plan ID)
- toasts: Array of notifications

DESIGN SYSTEM
- Uses existing BYOS design system
- Dark theme (bg-gray-950, gray-900)
- .card class for containers
- .btn-primary, .btn-secondary for buttons
- Tailwind utilities for layout
- Responsive grid: 1 column mobile, 3 columns desktop

AUTHENTICATION
- Bearer token auto-included via axios interceptor
- Token from localStorage.getItem('byos_token')
- 401 errors redirect to /login (interceptor handles)
- No auth code in component itself

PERFORMANCE
- Minimal re-renders via useCallback deps
- Efficient polling with cleanup on unmount
- Promise.all for parallel API requests
- No unnecessary state updates

BROWSER SUPPORT
- ES6+ JavaScript
- CSS Grid and Flexbox
- LocalStorage API
- Modern fetch/axios
- Mobile-responsive (tested at 320px+)
