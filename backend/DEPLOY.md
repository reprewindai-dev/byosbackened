# BYOS Backend — Deployment Guide

This file gets you from "code on a laptop" to "production-grade endpoint serving thousands of concurrent users with honest sub-second P95."

Time to deploy: **~30 minutes**.
Monthly cost: **~$50–80** (Render Standard tier).
Result: **Postgres + Redis + 4-worker gunicorn behind HTTPS**, ready to plug Cloudflare in front for global edge caching.

---

## Option A — Render (recommended; Blueprint deploy)

### 1. Push to GitHub

```bash
git add .
git commit -m "Production-ready deploy config"
git push origin main
```

### 2. Create Render Blueprint

1. Go to **https://dashboard.render.com/blueprints**
2. Click **New Blueprint Instance**
3. Connect the GitHub repo
4. Render reads `render.yaml` and proposes:
   - `byos-ai-db` — Postgres Standard ($20/mo)
   - `byos-ai-redis` — Redis Starter ($10/mo)
   - `byos-ai-api` — Web Standard ($25/mo, 2 CPU / 2 GB)
   - `byos-ai-worker` — Worker Starter ($7/mo)
5. Click **Apply**. First deploy takes ~5–8 min (Docker build + Postgres provision).

### 3. Set the secrets that `render.yaml` left blank

In the API service → **Environment** tab, set:

| Var | Where to get it |
|---|---|
| `OPENAI_API_KEY` | https://platform.openai.com/api-keys |
| `HUGGINGFACE_API_KEY` | https://huggingface.co/settings/tokens |
| `STRIPE_SECRET_KEY` | https://dashboard.stripe.com/apikeys |
| `STRIPE_WEBHOOK_SECRET` | https://dashboard.stripe.com/webhooks |
| `S3_ACCESS_KEY_ID` / `S3_SECRET_ACCESS_KEY` | AWS S3, R2, or DigitalOcean Spaces |
| `AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY` / `AWS_DEFAULT_REGION` | Amazon Bedrock + Hetzner backup upload |
| `SENTRY_DSN` | optional, https://sentry.io |
| `CORS_ORIGINS` | your real frontend URL, e.g. `["https://app.yourcompany.com"]` |

`SECRET_KEY` and `ENCRYPTION_KEY` are auto-generated by Render (`generateValue: true` in `render.yaml`).

Click **Save Changes** — Render redeploys automatically.

### 4. Run database migrations (one-time)

In the API service → **Shell** tab:

```bash
alembic upgrade head
```

### 5. Verify

```bash
curl https://byos-ai-api.onrender.com/health
# {"status":"ok","version":"0.1.0","service":"BYOS AI Backend"}

curl -I https://byos-ai-api.onrender.com/health
# HTTP/2 200
# x-fast-path: l1
# cache-control: public, max-age=30, s-maxage=60
```

The `x-fast-path: l1` header proves the fast-path middleware is active.

## Option B — DigitalOcean App Platform

1. **App Platform → Create App → GitHub**
2. Detect Dockerfile at `infra/docker/Dockerfile.api`
3. Add managed Postgres (cluster, $15/mo) and Redis ($15/mo)
4. Set env vars (same list as Render, see `.env.production.example`)
5. Set `WEB_CONCURRENCY=4`, plan = Basic ($12) or Pro ($24)
6. Deploy.

## Option C — Self-host on any Linux VM

```bash
# 1. Provision Postgres + Redis (managed or local)
# 2. Clone repo
git clone <your-repo> && cd backend

# 3. Create .env from template
cp .env.production.example .env.production
$EDITOR .env.production   # fill in DATABASE_URL, REDIS_URL, secrets

# 4. Build and run
docker build -f infra/docker/Dockerfile.api -t byos-api .
docker run -d --name byos-api --env-file .env.production -p 8000:8000 byos-api

# 5. Migrate
docker exec byos-api alembic upgrade head

# 6. Put nginx + Let's Encrypt in front (or use Caddy for auto-TLS)
```

For the production stack in `docker-compose.prod.yml`, Ollama now runs as a named server container:

```bash
docker compose -f docker-compose.prod.yml up -d
docker compose -f docker-compose.prod.yml exec ollama ollama pull qwen2.5:1.5b
docker compose -f docker-compose.prod.yml exec api alembic upgrade head
```

That keeps the public site pointed at `http://ollama:11434` inside the server network instead of back at your laptop.

## After deploy: turn on Cloudflare for free 10× scale on public endpoints

1. Add the domain to Cloudflare (free plan is enough)
2. Point DNS at your Render/DO/VM IP
3. **Rules → Page Rules → Create**:
   - URL: `your-api.com/health` → Cache Level: Cache Everything, Edge TTL: 30s
   - URL: `your-api.com/api/v1/openapi.json` → same
   - URL: `your-api.com/api/v1/docs` → same
4. Enable **Auto Minify**, **Brotli**, **HTTP/3**
5. Done. `/health`, `/openapi.json`, `/docs` now served from Cloudflare's 300+ edge POPs in <50 ms globally, **without ever hitting your origin** for cache hits.

The `Cache-Control: public, s-maxage=60` header that `FastPathMiddleware` already emits is what tells Cloudflare to cache.

## Troubleshooting

**Health check failing on first deploy:**
- Check service logs. Most common: `DATABASE_URL` not yet provisioned. Wait 1 min, redeploy.

**Slow cold starts:**
- Render Starter spins down after inactivity. Move API to Standard plan (always-on).

**429 errors under load:**
- Per-IP rate limit kicked in. Check `_DEFAULT_IP_LIMIT` in `apps/api/middleware/rate_limit.py`. With Cloudflare in front, set `forwarded_allow_ips = "*"` (already in `gunicorn_conf.py`) and trust `X-Forwarded-For`.

**Redis connection errors in logs:**
- App fails open (caching disabled, rate limits become per-process). Check `REDIS_URL` is set and Redis service is healthy.

**`InvalidRequestError: pysqlite is not async`:**
- You forgot to set `DATABASE_URL` to a real Postgres URL. The async engine is conditional on URL scheme.
