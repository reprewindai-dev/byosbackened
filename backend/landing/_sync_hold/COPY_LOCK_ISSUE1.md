1. Live demo CTA (primary)
2. Marketplace access/login CTA (secondary)
3. API/backend value visibility
4. Vendor/GitHub listings visibility
5. Managed control-node deployment (sales-led)
6. Sovereign licensing / enterprise evaluation (sales-led)

## Do-not-say boundaries

The public `.com` surface must not:

- Position Veklom as generic app hosting
- Position Veklom as a cloud-provider panel
- Lead with acquisition/procurement-only framing in primary conversion zones
- Downscope Veklom into infra-only language that hides the control-plane value

- `.com` = showpiece + live demo + marketplace entry
- `.dev` = acquisition/strategic surface (when deployed)

Acquisition language stays implicit/secondary on `.com` and is never the top-of-page conversion objective.
