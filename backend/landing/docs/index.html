Architecture

Veklom is a FastAPI, PostgreSQL, Redis, Celery, Stripe, and local-LLM control plane. The runtime is designed for private VPC, on-prem, or air-gapped operation where the buyer needs operational evidence and source-level control.

Control Plane

API middleware enforces zero-trust checks, metrics, intelligent routing, budget controls, and tenant isolation before execution reaches protected business routers.

Data Plane

Local Ollama/vLLM-style execution is primary. External providers are explicit operator choices through configured provider keys and routing policy.

Tenant Isolation

Workspace ownership is enforced at the app layer and reinforced with Postgres row-level security on tenant-scoped execution records.

Operations

Prometheus-style metrics, structured logs, health endpoints, job history, alert surfaces, and budget controls provide the operating loop.

Authentication

Protected routes use bearer JWTs for user sessions and scoped API keys for machine-to-machine execution. Workspaces carry roles, entitlements, billing status, and usage limits.

Governed Inference

Inference is routed through policy, budget, cost, and audit layers. Circuit-breaker logic can move from local inference to an approved fallback only when the operator configures it.

Circuit Breaker

Redis-backed state tracks closed, open, and half-open recovery behavior so local model failures degrade through a controlled policy path.

Cost Controls

Workspace budgets, hard caps, forecast signals, and kill-switch behavior are enforced before runaway token spend becomes an invoice problem.

Tamper-Evident Audit

Every governed execution can emit a canonical audit record with content hashes, HMAC signature, and previous-log hash linkage. This gives regulated teams an evidence trail that can be verified after the fact.

Privacy And Compliance

The platform includes privacy workflows for access/export, deletion, PII detection, and masking. Compliance surfaces are designed around HIPAA, SOC 2, PCI-DSS, GDPR, and internal audit evidence.

DSAR Workflows

  • Article 15 export path
  • Article 17 erasure path
  • PII detection and masking

Compliance Evidence

  • Signed audit exports
  • Control mapping packages
  • Runtime policy decisions

Billing And Entitlements

Veklom supports subscription plans, usage limits, token or credit-style metering, Stripe checkout, webhook reconciliation, and entitlement-gated access to premium surfaces.

Revenue path: billing is tied to workspace access, API-key limits, execution quotas, support tier, and compliance package access.

Deployment Posture

The public landing app is static on Cloudflare Pages. The backend remains source of truth for API behavior at https://api.veklom.com/api/v1. Production runtime secrets must stay in environment variables or secret stores, never in public assets.

Private Deployment

VPC, Kubernetes, on-prem, and air-gapped deployment modes are supported by the architecture and documentation set.

Operational Boundary

Customer runtime data stays inside the customer boundary unless the operator explicitly enables an external provider route.

Workspace Routes
