Architecture

How It Works

01

Deploy Locally

Run .\start.ps1 — brings up Postgres, Redis, MinIO, and the API. Ollama runs natively on Windows with your GPU or CPU.

02

Authenticate via API Key

Seed a byos_ key, attach it as X-API-Key. Every request is tenant-resolved and RLS-enforced automatically.

03

Call POST /v1/exec

Send a prompt with optional conversation_id. The backend injects history, calls Ollama, and falls back to Groq if the circuit opens.

04

Everything Logged

Every execution stored in execution_logs with tenant, model, tokens, latency, and provider. Query it, export it, bill from it.
